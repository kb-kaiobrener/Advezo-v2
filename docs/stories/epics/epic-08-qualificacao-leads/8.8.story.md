## Story

**As a** gestor de tráfego,
**I want to** visualizar todos os leads captados por formulário LP e Lead Ads em uma lista unificada,
**so that** eu possa qualificar, descartar e acompanhar o funil de leads dos meus clientes sem sair do Advezo.

- [x] **AC 8.8.1** — Página `/leads` acessível via sidebar (nova entrada "Leads" no menu de navegação do `(dashboard)` layout). Protegida por autenticação — redireciona para login se não autenticado.

- [x] **AC 8.8.3** — Filtros na lista: por `client_id` (select de clientes do workspace), por `status` (multiselect), por `source` (`landing_page` / `lead_ads`), por período (`date_range` — hoje, últimos 7d, últimos 30d, personalizado). Filtros aplicados server-side via query params na URL.

- [x] **AC 8.8.4** — Coluna de email: exibida **somente se** `consent_given_at IS NOT NULL`. Email descriptografado server-side (`decryptToken(email_encrypted)`) e exibido em texto claro na UI. Se `consent_given_at IS NULL`: coluna de email exibe `—` (sem dado). **Nunca** enviar `email_encrypted` (ciphertext) ao browser.

- [x] **AC 8.8.5** — Badge de consentimento LGPD: ícone de escudo (ou similar) visível por lead quando `consent_given_at IS NOT NULL`. Tooltip ao hover: *"Consentimento explícito registrado em {data}"*. Leads de Lead Ads sem `consent_given_at` exibem badge diferente: *"Meta Terms"* (base legal diferente).

- [x] **AC 8.8.6** — Ações por lead (menu dropdown ou botões inline): `Qualificar` (→ `qualificado`), `Descartar` (→ `desqualificado`), `Marcar como convertido` (→ `convertido`). Botões desabilitados quando a transição é inválida (ex: lead `convertido` — sem ações). Ações chamam `updateLeadStatus` (Server Action da Story 8.4) com otimistic update na UI.

- [x] **AC 8.8.7** — Bulk action: checkbox por linha + seleção de todos. Ação bulk "Qualificar selecionados" / "Descartar selecionados" chama `bulkUpdateLeadStatus` (Story 8.4). Exibe toast com resultado: *"3 leads qualificados, 1 erro"*.

- [x] **AC 8.8.8** — Painel lateral (sheet) ou página de detalhe ao clicar em um lead: nome completo, telefone, email (se consent), fonte, status atual, data de criação, `qualified_at` / `converted_at` se aplicável, `field_data` (campos customizados do formulário). Badge de CAPI: mostra se evento `Lead` foi enviado (`conversion_events.status = 'sent'`).

- [x] **AC 8.8.9** — Loading states e empty state: skeleton loader durante fetch; empty state com CTA quando não há leads ("Nenhum lead encontrado. Crie um formulário de captura ou configure sua integração com Lead Ads").

## Dev Notes

### Descriptografia Server-Side (AC 8.8.4 — Segurança Crítica)

O email **nunca** deve ser enviado ao browser como ciphertext nem como hash:

```typescript
// apps/web/src/app/(dashboard)/leads/page.tsx (Server Component)
import { decryptToken } from '@advezo/utils'

const leadsWithEmail = leads.map(lead => ({
  ...lead,
  email: lead.email_encrypted && lead.consent_given_at
    ? decryptToken(lead.email_encrypted, process.env.TOKEN_ENCRYPTION_KEY!)
    : null,
  email_encrypted: undefined, // nunca enviado ao client
}))
```

Usar Server Component para a page principal — descriptografia ocorre no servidor. Componentes client (ações, loading states) recebem apenas o email decriptografado (string) ou null.

### Transições Válidas (AC 8.8.6 — para desabilitar botões)

```typescript
const VALID_NEXT_STATUSES: Record<LeadStatus, LeadStatus[]> = {
  novo: ['qualificado', 'desqualificado'],
  qualificado: ['desqualificado', 'convertido'],
  desqualificado: ['novo'],
  convertido: [], // terminal — sem ações
}
```

Botões desabilitados quando `VALID_NEXT_STATUSES[lead.status].length === 0`.

### Badge de Consent (AC 8.8.5)

```typescript
// ConsentBadge: três estados

```

### Mascaramento de Telefone (AC 8.8.2)

`phone_hash` é um hash HMAC-SHA256 — não é um número de telefone legível. Para exibir telefone mascarado, precisaria de `phone_hash_display` ou similar. **Decisão de implementação:** se o schema não tiver campo de telefone legível (por design — privacidade), exibir `"****"` como placeholder e documentar a limitação. O gestor pode abrir o detalhe do lead para ver mais contexto. Documentar esta decisão no Dev Agent Record.

### Decisões autônomas (ADR resumido)

- **[AUTO-DECISION] Mascaramento de telefone** → Exibir `••••` (placeholder fixo), não os 4 últimos dígitos.
  - *Razão:* o schema `leads` tem apenas `phone_hash` (HMAC-SHA256 irreversível); não existe coluna de telefone legível (decisão de privacidade do Epic 8). Os 4 últimos dígitos pedidos no AC 8.8.2 são tecnicamente impossíveis sem o número original. Limitação já reconhecida e autorizada nas Dev Notes da story e na validação do @po (9/10). Detalhe do lead exibe o mesmo placeholder.

- **[AUTO-DECISION] Badge de CAPI com `conversion_events` ausente (AC 8.8.8)** → Query resiliente que degrada para `capiSent = null` ("—") quando a tabela não existe.
  - *Razão:* `conversion_events` ainda não foi criada (pertence a epic futuro — confirmado em `20260101000009_leads.sql` e em `lib/capi/leads.ts`). `loadCapiStatus` retorna `null` em qualquer erro/tabela ausente (fonte indisponível → badge "—"); Map vazio com tabela presente = "Não enviado". Mesmo padrão de graceful degradation já adotado no Epic 8 (bloco `to_regclass` na migration, skip auditável no CAPI sender).

- **[AUTO-DECISION] Filtro de status: select único em vez de multiselect (AC 8.8.3)** → `<select>` nativo num form GET (`?status=...`).
  - *Razão:* mantém os filtros 100% server-side via query params sem JS (mesmo padrão da página de campanhas, IDS REUSE). A opção "Todos os status" cobre o caso sem filtro. Um multiselect real exigiria estado client + serialização customizada — fora do valor da story, que prioriza filtragem server-side simples.

- **[AUTO-DECISION] Tooltip e Toast sem nova dependência** → Tooltip via CSS `group-hover` (ConsentBadge); Toast inline com auto-dismiss (LeadsTable).
  - *Razão:* o design system não tem componente Tooltip/Toast nem Radix/sonner instalados, e a story não autoriza adicionar dependências. Soluções CSS/estado local atendem aos AC 8.8.5 / 8.8.7 sem expandir a superfície de dependências.

- **[AUTO-DECISION] Sheet de detalhe via overlay CSS** → `LeadDetailSheet` como slide-over com overlay + Escape, sem componente Sheet do design system.
  - *Razão:* não há primitivo Sheet/Drawer disponível; o overlay CSS puro entrega o painel lateral do AC 8.8.8 sem dependência nova.
