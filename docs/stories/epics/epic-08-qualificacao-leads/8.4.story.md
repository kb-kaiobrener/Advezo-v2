## Dev Notes

### Motor de Qualificação

```typescript
// apps/web/src/lib/leads/qualification.ts
import { Lead, QualificationRule } from '@advezo/types'

export function evaluateQualificationRules(
  fieldData: Record<string, unknown>,
  rules: QualificationRule[]
): boolean {
  if (!rules || rules.length === 0) return false

  return rules.every(rule => {
    const fieldValue = fieldData[rule.field]

    switch (rule.operator) {
      case 'eq':
        return String(fieldValue) === String(rule.value)
      case 'not_eq':
        return String(fieldValue) !== String(rule.value)
      case 'contains':
        return typeof fieldValue === 'string' && fieldValue.includes(rule.value ?? '')
      case 'filled':
        return fieldValue !== undefined && fieldValue !== null && fieldValue !== ''
      case 'not_filled':
        return fieldValue === undefined || fieldValue === null || fieldValue === ''
      default:
        return false
    }
  })
}
```

  'novo': ['qualificado', 'desqualificado'],

### Server Action updateLeadStatus

```typescript
// apps/web/src/app/actions/leads.ts
'use server'
import { revalidatePath } from 'next/cache'
import { createSupabaseServerClient } from '@advezo/database'
import { LeadStatus } from '@advezo/types'

export async function updateLeadStatus(leadId: string, newStatus: LeadStatus) {
  const supabase = createSupabaseServerClient()
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) return { error: 'Unauthorized' }

  // Buscar lead atual para validar transição
  const { data: lead } = await supabase
    .from('leads')
    .select('status, source, consent_given_at, phone_hash, email_encrypted, client_id')
    .eq('id', leadId)
    .single()

  if (!lead) return { error: 'Lead não encontrado' }
  if (lead.status === 'convertido') return { error: 'status_convertido_terminal' }

  const allowed = VALID_TRANSITIONS[lead.status]
  if (!allowed.includes(newStatus)) {
    return { error: `Transição ${lead.status}→${newStatus} não permitida` }
  }

  const updateData: Record<string, unknown> = {
    status: newStatus,
    updated_at: new Date().toISOString(),
  }
  if (newStatus === 'qualificado') updateData.qualified_at = new Date().toISOString()
  if (newStatus === 'convertido') updateData.converted_at = new Date().toISOString()

  const { error } = await supabase.from('leads').update(updateData).eq('id', leadId)
  if (error) return { error: error.message }

  // Disparo CAPI assíncrono
  if (newStatus === 'qualificado') {
    sendCompleteRegistrationCapi(lead, leadId).catch(console.error)
  }
  if (newStatus === 'convertido') {
    sendPurchaseCapi(lead, leadId).catch(console.error)
  }

  revalidatePath('/leads')
  revalidatePath('/dashboard')
  return {}
}
```

### Índice Parcial e Desqualificação (FR-QC6)

Quando lead passa para `status='desqualificado'`, a linha deixa de satisfazer o predicado `WHERE status NOT IN ('desqualificado')` do `leads_active_dedup`. O slot é liberado imediatamente — nenhuma lógica adicional necessária. Próxima campanha pode capturar o mesmo (client_id, phone_hash).

### Segurança

- `email_encrypted`: nunca logado. Descriptografar apenas em memória para envio CAPI.
- `SHA256(email)`: calculado em memória para CAPI — nunca persistido.
- Server Actions protegidas por `supabase.auth.getUser()` — sem autenticação → erro, não 401 HTTP direto (padrão Next.js Server Actions).

## Testing

- **Operador eq:** `{ field: 'q1', operator: 'eq', value: 'Sim' }` + `field_data: { q1: 'Sim' }` → true
- **Operador not_eq:** `value: 'Não'` + `field_data: { q1: 'Sim' }` → true
- **Operador contains:** `{ field: 'msg', operator: 'contains', value: 'urgente' }` + `field_data: { msg: 'Preciso urgente' }` → true
- **Operador filled:** campo presente e não vazio → true; campo vazio → false
- **AND-logic:** 2 regras, primeira passa e segunda falha → false
- **Array vazio:** `evaluateQualificationRules(fieldData, [])` → false (sem qualificação automática)
- **Transição inválida:** `updateLeadStatus(convertido_lead, 'qualificado')` → `{ error: 'status_convertido_terminal' }`
- **Transição válida:** `updateLeadStatus(novo_lead, 'qualificado')` → `{}` + revalidatePath chamado
- **bulkUpdate:** 3 leadIds, 1 falha → `{ updated: 2, errors: ['...'] }`
