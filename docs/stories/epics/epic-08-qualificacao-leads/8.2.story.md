## Story

**As a** gestor de tráfego,
**I want to** criar e gerenciar formulários de captura de leads com campos configuráveis e receber um snippet JS pronto para embedar,
**so that** eu possa instalar formulários nas landing pages dos meus clientes sem esforço técnico.

## Acceptance Criteria

- [x] **AC 8.2.1** — CRUD completo de `lead_forms` via API routes autenticadas por JWT: `GET /api/lead-forms` (lista por workspace), `POST /api/lead-forms` (cria), `GET /api/lead-forms/:id` (detalhe), `PUT /api/lead-forms/:id` (atualiza), `DELETE /api/lead-forms/:id` (soft delete: `is_active = false`). Todas as rotas protegidas por `supabase.auth.getUser()` — retorna 401 se não autenticado.

- [x] **AC 8.2.2** — Na criação (`POST /api/lead-forms`): `embed_token` gerado automaticamente com `crypto.randomBytes(16).toString('base64url')` — nunca exposto em logs. Slug gerado a partir do nome (lowercase, kebab-case, único por workspace — sufixo aleatório em caso de colisão). `client_id` obrigatório no body.

- [x] **AC 8.2.3** — Validação server-side do campo `fields` (jsonb): se `fields` contém algum item com `type: 'email'`, deve conter exatamente um item com `type: 'consent_checkbox'` e `linked_field: 'email'`. Caso contrário, retornar 422 com mensagem clara: `"Formulário com campo email requer consent_checkbox vinculado (LGPD Art. 7º I)"`. Esta validação aplica tanto no `POST` quanto no `PUT`.

- [x] **AC 8.2.4** — `GET /api/lead-forms/:id/embed` retorna objeto JSON com: `snippet` (string HTML do script tag pronto para copiar), `embed_token`, e `instructions` (texto curto com instruções de uso). Snippet format: `<script src="https://app.advezo.com.br/embed/form.js?token={embed_token}"></script>`.

- [x] **AC 8.2.5** — Tipos TypeScript: `CreateLeadFormInput`, `UpdateLeadFormInput`, `LeadFormEmbedResponse` adicionados em `packages/types/src/index.ts`. Validação Zod criada em `apps/web/src/lib/validation/lead-forms.ts` (reutilizada nas API routes).

## Dev Notes

### Geração de embed_token

```typescript
import { randomBytes } from 'crypto'

// SEGURANÇA: nunca sequencial, nunca derivado do slug, sempre 128 bits
const embed_token = randomBytes(16).toString('base64url')
```

`embed_token` é único por constraint `UNIQUE` na tabela. Em caso de colisão extremamente improvável (128 bits), repetir geração (loop com máximo 3 tentativas antes de erro 500).

### Geração de slug

```typescript
function generateSlug(name: string): string {
  return name
    .toLowerCase()
    .normalize('NFD')
    .replace(/[̀-ͯ]/g, '') // remove acentos
    .replace(/[^a-z0-9]+/g, '-')
    .replace(/^-|-$/g, '')
}
// Em caso de colisão com workspace_slug UNIQUE: append de 4 chars aleatórios
// Ex: 'meu-formulario-a3f2'
```

### Validação Zod — regra consent_checkbox

```typescript
// apps/web/src/lib/validation/lead-forms.ts
import { z } from 'zod'

const fieldSchema = z.object({
  id: z.string(),
  type: z.enum(['text', 'phone', 'email', 'consent_checkbox', 'select', 'boolean']),
  label: z.string(),
  required: z.boolean(),
  fixed: z.boolean().optional(),
  linked_field: z.string().optional(),
  options: z.array(z.string()).optional(),
})

export const leadFormFieldsSchema = z.array(fieldSchema).superRefine((fields, ctx) => {
  const hasEmail = fields.some(f => f.type === 'email')
  const hasConsentForEmail = fields.some(
    f => f.type === 'consent_checkbox' && f.linked_field === 'email'
  )
  if (hasEmail && !hasConsentForEmail) {
    ctx.addIssue({
      code: z.ZodIssueCode.custom,
      message: 'Formulário com campo email requer consent_checkbox vinculado (LGPD Art. 7º I)',
    })
  }
})
```
