## Story

**As a** gestor de tráfego,
**I want to** conectar um número de WhatsApp a cada cliente,
**so that** relatórios e alertas sejam enviados pelo número correto sem misturar comunicações entre clientes.

## Acceptance Criteria

- [ ] **AC 3.2.1** — UI em Clientes → [cliente] → Configurações: seção "WhatsApp" com botão "Conectar WhatsApp" que abre dialog/modal com QR code gerado pelo worker Baileys. QR code exibido como imagem e atualizado automaticamente (polling a cada 2s) até escaneamento ou timeout de 5 minutos.

- [ ] **AC 3.2.2** — Após escaneamento confirmado (worker retorna status `connected`), registro salvo/atualizado em `whatsapp_connections`: `workspace_id`, `client_id`, `account_id` (E.164 do número conectado), `status='connected'`, `connected_at=now()`.

- [ ] **AC 3.2.3** — Campo "Template de aviso ao titular" editável na UI (textarea): texto enviado automaticamente ao lead na primeira conversa rastreada (NFR-8). Valor salvo em `whatsapp_connections.notice_template`. Campo opcional — pode ficar em branco.

- [ ] **AC 3.2.4** — Status de conexão visível na UI por número, com 4 estados distintos:
  - ✅ **Conectado** (verde): `whatsapp_connections.status = 'connected'`
  - ⏳ **Aguardando QR** (amarelo): `whatsapp_connections.status = 'connecting'` ou QR em polling
  - ❌ **Desconectado** (vermelho): `whatsapp_connections.status = 'disconnected'`
  - ⚠️ **Pausado — Circuit Breaker** (vermelho escuro + tooltip explicativo): `whatsapp_accounts.cb_paused_at IS NOT NULL`

- [ ] **AC 3.2.5** — Suporte a múltiplos números por cliente: um cliente pode ter N conexões WhatsApp (N rows em `whatsapp_connections` com mesmo `client_id`, `account_id` distintos). UI lista todas as conexões com status individual e botão de ação por conexão.

- [ ] **AC 3.2.6** — Desconexão manual: botão "Desconectar" por conexão na UI chama `POST /disconnect` no worker (fecha socket Baileys e faz logout) e atualiza `whatsapp_connections.status = 'disconnected'`. Após desconexão, botão muda para "Conectar" novamente.

- [ ] **AC 3.2.7** — Reconexão automática preservada: ao reiniciar o worker Railway, sessão é restaurada do Supabase Storage e `whatsapp_accounts.status` é atualizado para `connected` sem intervenção manual. A UI reflete o status correto ao recarregar (leitura do banco, não do worker diretamente).

- [ ] **AC 3.2.8** — Circuit breaker visível: quando `whatsapp_accounts.cb_paused_at IS NOT NULL`, UI exibe badge "⚠️ Pausado" com tooltip "Muitas falhas de reconexão — número pausado temporariamente". Botão "Resetar" disponível: limpa `cb_paused_at` e `cb_failure_count` no banco e tenta reconectar.

### Relação entre whatsapp_accounts e whatsapp_connections

```
whatsapp_accounts      ← worker-managed (Baileys session, circuit breaker)
  workspace_id
  account_id (PK natural: ws+acc)
  status, cb_failure_count, cb_paused_at

whatsapp_connections   ← client-facing mapping (esta story)
  workspace_id
  client_id            ← liga ao cliente
  account_id           ← JOIN com whatsapp_accounts por (workspace_id, account_id)
  status               ← espelho do status real (atualizado pelo worker via markDisconnected)
  notice_template
```

Para exibir o status completo (incluindo `cb_paused_at`), a UI faz join:

### Worker routes novas (T3, T4)

```typescript
// GET /status?workspace_id=&account_id=
app.get('/status', (req: Request, res: Response) => {
  const workspaceId = String(req.query.workspace_id ?? '')
  const accountId   = String(req.query.account_id ?? '')
  if (!workspaceId || !accountId) {
    res.status(400).json({ error: 'workspace_id e account_id são obrigatórios' })
    return
  }
  const key = accountKey(workspaceId, accountId)
  const hasSocket     = sockets.has(key)
  const isConnecting  = connectingKeys.has(key)
  const hasQr         = qrCache.has(key)

  let status: 'connected' | 'connecting' | 'disconnected'
  if (hasSocket && !hasQr) status = 'connected'
  else if (isConnecting || hasQr) status = 'connecting'
  else status = 'disconnected'

  res.status(200).json({ status })
})

// POST /disconnect
app.post('/disconnect', async (req: Request, res: Response) => {
  const { workspace_id, account_id } = req.body as {
    workspace_id?: string
    account_id?: string
  }
  if (!workspace_id || !account_id) {
    res.status(400).json({ error: 'workspace_id e account_id são obrigatórios' })
    return
  }
  const key = accountKey(workspace_id, account_id)
  const sock = sockets.get(key)
  if (sock) {
    try {
      await sock.logout()
    } catch (err) {
      logger.error({ err, account_id }, 'falha no logout Baileys — removendo socket mesmo assim')
    }
    sockets.delete(key)
    qrCache.delete(key)
    connectingKeys.delete(key)
  }
  try {
    await markDisconnected(workspace_id, account_id)
  } catch (err) {
    logger.error({ err, account_id }, 'falha ao marcar desconectado no banco')
  }
  res.status(200).json({ ok: true })
})
```

### Server Actions (T5)

```typescript
// apps/web/src/app/actions/whatsapp.ts
'use server'
import { createSupabaseServerClient, createSupabaseServiceClient } from '@advezo/database'

const WORKER_URL = process.env.WHATSAPP_WORKER_URL!

export async function connectWhatsApp(clientId: string, accountId: string) {
  const supabase = await createSupabaseServerClient()
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) return { error: 'Não autenticado' }

  const serviceClient = createSupabaseServiceClient()
  const { data: membership } = await serviceClient
    .from('workspace_members').select('workspace_id').eq('user_id', user.id).limit(1).single()
  if (!membership) return { error: 'Workspace não encontrado' }

  const { error } = await serviceClient
    .from('whatsapp_connections')
    .upsert({
      workspace_id: membership.workspace_id,
      client_id: clientId,
      account_id: accountId,
      status: 'connecting',
    }, { onConflict: 'workspace_id,client_id,account_id' })

  if (error) return { error: 'Erro ao salvar conexão' }
  return { success: true, workspaceId: membership.workspace_id }
}

export async function confirmWhatsAppConnected(clientId: string, accountId: string) {
  const supabase = await createSupabaseServerClient()
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) return { error: 'Não autenticado' }

  const serviceClient = createSupabaseServiceClient()
  const { data: membership } = await serviceClient
    .from('workspace_members').select('workspace_id').eq('user_id', user.id).limit(1).single()
  if (!membership) return { error: 'Workspace não encontrado' }

  const { error } = await serviceClient
    .from('whatsapp_connections')
    .update({ status: 'connected', connected_at: new Date().toISOString() })
    .eq('workspace_id', membership.workspace_id)
    .eq('client_id', clientId)
    .eq('account_id', accountId)

  if (error) return { error: 'Erro ao confirmar conexão' }
  return { success: true }
}

export async function disconnectWhatsApp(clientId: string, accountId: string) {
  const supabase = await createSupabaseServerClient()
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) return { error: 'Não autenticado' }

  const serviceClient = createSupabaseServiceClient()
  const { data: membership } = await serviceClient
    .from('workspace_members').select('workspace_id').eq('user_id', user.id).limit(1).single()
  if (!membership) return { error: 'Workspace não encontrado' }

  // Chama o worker para fechar o socket
  try {
    await fetch(`${WORKER_URL}/disconnect`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ workspace_id: membership.workspace_id, account_id: accountId }),
    })
  } catch (err) {
    // Worker pode estar offline — atualiza status no banco mesmo assim
    console.error('Worker disconnect falhou:', err)
  }

  const { error } = await serviceClient
    .from('whatsapp_connections')
    .update({ status: 'disconnected', connected_at: null })
    .eq('workspace_id', membership.workspace_id)
    .eq('client_id', clientId)
    .eq('account_id', accountId)

  if (error) return { error: 'Erro ao atualizar status' }
  return { success: true }
}

export async function resetCircuitBreaker(accountId: string) {
```

### QR Polling Flow (UI — T7)

O componente `QrPollingDialog` usa o endpoint `GET /qr` do worker via rotas de proxy autenticadas do Next.js (criadas em T10), que validam a sessão do usuário e o workspace antes de repassar a chamada — `WHATSAPP_WORKER_URL` nunca é exposta ao browser.

```
/api/whatsapp/qr?workspace_id=&account_id=     (Next.js route → valida sessão → proxy → worker GET /qr)
/api/whatsapp/status?workspace_id=&account_id= (Next.js route → valida sessão → proxy → worker GET /status)
```

Estados do polling:
1. **waiting**: QR ainda não gerado (worker retorna 202) → spinner
2. **scanning**: QR disponível → exibe `<img src={qr} />`
3. **connected**: QR desapareceu + `/api/whatsapp/status` retorna `connected` → ✅ + fecha dialog + chama `confirmWhatsAppConnected(clientId, accountId)` para gravar `status='connected'` e `connected_at=now()` em `whatsapp_connections`
4. **timeout**: 5 minutos sem conexão → erro "QR code expirado. Tente novamente."

### Rotas de proxy autenticado Next.js (T10)

Criadas em `apps/web/src/app/api/whatsapp/`. Cada rota valida a sessão do usuário e confirma que `workspace_id` pertence ao workspace do usuário antes de repassar ao worker. `WHATSAPP_WORKER_URL` é server-only — nunca exposta ao browser via `NEXT_PUBLIC_`.

```typescript
// apps/web/src/app/api/whatsapp/qr/route.ts
import { NextRequest, NextResponse } from 'next/server'
import { createSupabaseServerClient, createSupabaseServiceClient } from '@advezo/database'

export async function GET(req: NextRequest) {
  const supabase = await createSupabaseServerClient()
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) return NextResponse.json({ error: 'Não autenticado' }, { status: 401 })

  const workspaceId = req.nextUrl.searchParams.get('workspace_id') ?? ''
  const accountId   = req.nextUrl.searchParams.get('account_id') ?? ''
  if (!workspaceId || !accountId)
    return NextResponse.json({ error: 'workspace_id e account_id são obrigatórios' }, { status: 400 })

  // Valida que o workspace_id pertence ao usuário
  const serviceClient = createSupabaseServiceClient()
  const { data: membership } = await serviceClient
    .from('workspace_members').select('workspace_id').eq('user_id', user.id).eq('workspace_id', workspaceId).limit(1).single()
  if (!membership) return NextResponse.json({ error: 'Acesso negado' }, { status: 403 })

  const url = `${process.env.WHATSAPP_WORKER_URL}/qr?workspace_id=${workspaceId}&account_id=${accountId}`
  const workerRes = await fetch(url)
  const data = await workerRes.json()
  return NextResponse.json(data, { status: workerRes.status })
}
```
