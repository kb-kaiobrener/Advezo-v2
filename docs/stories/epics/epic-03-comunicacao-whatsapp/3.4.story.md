## Story

**As a** sistema do Advezo,
**I want to** gerar automaticamente o texto do relatório com as métricas do período, adaptado ao objetivo predominante da conta,
**so that** o conteúdo seja correto e formatado antes de ser enviado via WhatsApp.

> Esta story é **lógica pura**: função de geração testável unitariamente sem conexão WhatsApp, sem chamadas a APIs externas durante a geração. Pode rodar em paralelo com Story 3.2.

## Acceptance Criteria

- [x] **AC 3.4.1** — Função `generateReport(params)` recebe `workspaceId`, `clientId`, `period: { from: Date, to: Date }` e retorna `Promise<string>` com o texto formatado. Exportada de `packages/report-generator/` (novo pacote) ou de `apps/web/src/lib/report-generator.ts` (se pacote separado for overkill — @dev decide IDS: REUSE/ADAPT/CREATE).

- [x] **AC 3.4.2** — Detecção automática do objetivo predominante pelas campanhas do cliente: lê `ad_campaigns.objective` (campo text) das campanhas do período e mapeia para 3 tipos de relatório:
  - **vendas** (`objective` contém `SALES`, `CONVERSIONS` ou `PURCHASE`) → exibe ROAS e conversões
  - **leads** (`objective` contém `LEAD`) → exibe CPL e volume de leads
  - **mensagens** (`objective` contém `MESSAGES`, `ENGAGEMENT`) → exibe CPM e volume
  - **default** (objetivo ausente ou não mapeado) → relatório genérico com spend + impressões + cliques

- [x] **AC 3.4.3** — Formato texto puro compatível com WhatsApp: sem HTML, sem Markdown de tabela, somente bold via `*texto*` (suportado pelo WhatsApp), emojis de status, números em PT-BR (`R$ 1.234,56`, `2,7%`, `4,6x`).

- [x] **AC 3.4.4** — Dados exclusivamente de `campaign_metrics` via Supabase (sem chamada à API Meta/Google durante geração). Agregação server-side: `SUM(spend)`, `SUM(impressions)`, `SUM(clicks)`, `SUM(conversions)`, `SUM(revenue)` no período `[from, to]` para as campanhas do cliente.

- [x] **AC 3.4.5** — 3 cenários de teste unitário obrigatórios (dados mockados, sem Supabase real):
  1. **Vendas:** campanha com `objective='OUTCOME_SALES'`, metrics com revenue > 0 → texto contém ROAS e conversões
  2. **Leads:** campanha com `objective='OUTCOME_LEADS'`, metrics com conversions > 0, revenue = 0 → texto contém CPL e volume de leads
  3. **Default/sem objetivo:** campanha com `objective=null`, metrics básicos → texto genérico com spend + impressões

```

  └─ ad_accounts (WHERE client_id = clientId AND workspace_id = workspaceId)
       └─ ad_campaigns (WHERE ad_account_id IN (...))
            └─ campaign_metrics (WHERE campaign_id IN (...) AND date BETWEEN from AND to)
```

### Implementação — função generateReport

```typescript
// apps/web/src/lib/report-generator.ts
import { createSupabaseServiceClient } from '@advezo/database'

interface Period { from: Date; to: Date }

export async function generateReport(
  workspaceId: string,
  clientId: string,
  period: Period
): Promise<string> {
  const db = createSupabaseServiceClient()

  // 1. Busca ad_accounts do cliente
  const { data: accounts } = await db
    .from('ad_accounts')
    .select('id')
    .eq('client_id', clientId)
    .eq('workspace_id', workspaceId)

  if (!accounts?.length) return buildEmptyReport(period)
  const accountIds = accounts.map(a => a.id)

  // 2. Busca campanhas com objective
  const { data: campaigns } = await db
    .from('ad_campaigns')
    .select('id, name, objective')
    .in('ad_account_id', accountIds)
    .eq('workspace_id', workspaceId)

  if (!campaigns?.length) return buildEmptyReport(period)
  const campaignIds = campaigns.map(c => c.id)

  // 3. Agrega métricas no período
  const fromStr = period.from.toISOString().split('T')[0]
  const toStr   = period.to.toISOString().split('T')[0]
  const { data: metrics } = await db
    .from('campaign_metrics')
    .select('campaign_id, impressions, clicks, spend, conversions, revenue')
    .in('campaign_id', campaignIds)
    .gte('date', fromStr)
    .lte('date', toStr)

  const totals = aggregateMetrics(metrics ?? [])

  // 4. Detecta objetivo predominante
  const objectives = campaigns.map(c => c.objective ?? '')
  const type = classifyObjectives(objectives)

  // 5. Formata texto
  return buildReport(type, totals, period)
}
```

### classifyObjective (AC 3.4.2)

```typescript
type ReportType = 'vendas' | 'leads' | 'mensagens' | 'default'

function classifyObjectives(objectives: string[]): ReportType {
  const joined = objectives.join(' ').toUpperCase()
  if (/SALES|CONVERSIONS|PURCHASE/.test(joined)) return 'vendas'
  if (/LEAD/.test(joined))                         return 'leads'
  if (/MESSAGES|MESSAGING|ENGAGEMENT/.test(joined)) return 'mensagens'
  return 'default'
}
```

### Formatadores PT-BR (AC 3.4.3)

```typescript
const ptBR = new Intl.NumberFormat('pt-BR', { style: 'currency', currency: 'BRL' })
const ptBRNum = new Intl.NumberFormat('pt-BR')
const ptBRPct = new Intl.NumberFormat('pt-BR', { style: 'percent', minimumFractionDigits: 1, maximumFractionDigits: 1 })

function formatBRL(n: number)         { return ptBR.format(n) }              // R$ 1.234,56
function formatNumber(n: number)      { return ptBRNum.format(n) }           // 45.678
function formatPercent(n: number)     { return ptBRPct.format(n) }           // 2,7%
function formatMultiplier(n: number)  { return `${n.toFixed(1).replace('.', ',')}x` } // 4,6x
```

### Templates de texto (AC 3.4.3 + 3.4.2)

```typescript
function buildReport(type: ReportType, t: Totals, period: Period): string {
  const header = formatPeriodHeader(period)

  switch (type) {
    case 'vendas': {
      const roas = t.spend > 0 ? t.revenue / t.spend : 0
      const ctr  = t.impressions > 0 ? t.clicks / t.impressions : 0
      return [
        `📊 *Relatório de Vendas*`,
        header,
        ``,
        `💰 *Investimento:* ${formatBRL(t.spend)}`,
        `📈 *Impressões:* ${formatNumber(t.impressions)}`,
        `👆 *Cliques:* ${formatNumber(t.clicks)} (CTR: ${formatPercent(ctr)})`,
        ``,
        `🎯 *Conversões:* ${formatNumber(t.conversions)}`,
        `💵 *Receita:* ${formatBRL(t.revenue)}`,
        `🏆 *ROAS:* ${formatMultiplier(roas)}`,
        ``,
        `_Gerado automaticamente pelo Advezo._`,
      ].join('\n')
    }
    case 'leads': {
      const cpl = t.conversions > 0 ? t.spend / t.conversions : 0
      const ctr  = t.impressions > 0 ? t.clicks / t.impressions : 0
      return [
        `📊 *Relatório de Leads*`,
        header,
        ``,
        `💰 *Investimento:* ${formatBRL(t.spend)}`,
        `📈 *Impressões:* ${formatNumber(t.impressions)}`,
        `👆 *Cliques:* ${formatNumber(t.clicks)} (CTR: ${formatPercent(ctr)})`,
        ``,
        `🎯 *Leads gerados:* ${formatNumber(t.conversions)}`,
        `💡 *CPL:* ${formatBRL(cpl)}`,
        ``,
        `_Gerado automaticamente pelo Advezo._`,
      ].join('\n')
    }
    case 'mensagens': {
      const cpm = t.impressions > 0 ? (t.spend / t.impressions) * 1000 : 0
      return [
        `📊 *Relatório de Mensagens*`,
        header,
        ``,
        `💰 *Investimento:* ${formatBRL(t.spend)}`,
        `📈 *Impressões:* ${formatNumber(t.impressions)}`,
        `💬 *Mensagens:* ${formatNumber(t.conversions)}`,
        `💡 *CPM:* ${formatBRL(cpm)}`,
        ``,
        `_Gerado automaticamente pelo Advezo._`,
      ].join('\n')
    }
    default: {
      const ctr = t.impressions > 0 ? t.clicks / t.impressions : 0
      return [
        `📊 *Relatório de Campanhas*`,
        header,
        ``,
        `💰 *Investimento:* ${formatBRL(t.spend)}`,
        `📈 *Impressões:* ${formatNumber(t.impressions)}`,
        `👆 *Cliques:* ${formatNumber(t.clicks)} (CTR: ${formatPercent(ctr)})`,
        ``,
        `_Gerado automaticamente pelo Advezo._`,
      ].join('\n')
    }
  }
}
```
