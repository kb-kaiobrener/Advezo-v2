### Arquitetura de cache e autenticação — decisão fechada

#### Por que `cookies()` no `page.tsx` não resolve

No App Router do Next.js sem PPR, chamar `cookies()` (de `next/headers`) em qualquer ponto do Server Component — mesmo dentro de um `if` — opta a **rota inteira** para dynamic rendering. A decisão é por rota, não por branch de código. Isso significa que um `if (config.password_hash) { cookies()... }` tornaria `/dashboard/[token]` dinâmica para todos os visitantes, inclusive dashboards sem senha, destruindo o ISR.

#### Arquitetura correta: middleware como gate

```
Visitante → GET /dashboard/:token
              ↓
         Middleware (Edge)
              ├── lê password_hash via Supabase REST (fetch leve)
              ├── se sem senha → NextResponse.next() (page.tsx serve ISR)
              └── se com senha:
                    ├── cookie válido → NextResponse.next()
                    └── sem cookie / inválido → redirect /dashboard/:token/senha

/dashboard/[token]/page.tsx     ← ISR puro, export const revalidate = 300, zero cookies()
/dashboard/[token]/senha/page.tsx ← PasswordGate, pode ser dinâmica, redireciona após auth
```

#### Middleware — proteção por senha (T5.5)

O middleware deve interceptar `/dashboard/:token` (sem o sufixo `/senha` para evitar loop) e:
1. Fazer fetch Supabase REST para buscar `password_hash` e `password_salt` — query leve, um campo
2. Se `password_hash = null` → `NextResponse.next()` (dashboard público)
3. Se `password_hash != null` → verificar cookie; se inválido → redirect `/senha`

**WebCrypto no Edge:** middleware roda no Edge runtime, sem acesso a `node:crypto`. Usar a WebCrypto API do browser (disponível em Edge Functions do Next.js via `globalThis.crypto`):

```typescript
// Verificação de cookie com WebCrypto (sem node:crypto)
async function verifyDashboardCookie(
  cookieValue: string,
  token: string,
  passwordHash: string,
  secret: string
): Promise<boolean> {
  const enc = new TextEncoder()
  const key = await crypto.subtle.importKey(
    'raw',
    enc.encode(secret),
    { name: 'HMAC', hash: 'SHA-256' },
    false,
    ['verify']
  )
  // Converter o valor hex do cookie de volta para bytes para usar crypto.subtle.verify
  const cookieBytes = Uint8Array.from(
    cookieValue.match(/../g)!.map(h => parseInt(h, 16))
  )
  const message = enc.encode(token + passwordHash)
  // crypto.subtle.verify é timing-safe por spec
  return crypto.subtle.verify('HMAC', key, cookieBytes, message)
}
```

### Proteção por senha — Route Handler

```typescript
// apps/web/src/app/api/dashboard/[token]/auth/route.ts
import { NextRequest, NextResponse } from 'next/server'
import crypto from 'node:crypto'

export async function POST(req: NextRequest, { params }: { params: { token: string } }) {
  const { password } = await req.json() as { password?: string }
  if (!password) return NextResponse.json({ error: 'senha obrigatória' }, { status: 400 })

  const db = createSupabaseServiceClient()
  const { data: config } = await db
    .from('dashboard_configs')
    .select('password_hash, password_salt')
    .eq('token', params.token)
    .eq('is_active', true)
    .single()

  if (!config?.password_hash) return NextResponse.json({ error: 'token inválido' }, { status: 404 })

  const secret = process.env.DASHBOARD_AUTH_SECRET!
  const expected = crypto.createHmac('sha256', secret)
    .update(password + config.password_salt)
    .digest('hex')

  // timingSafeEqual previne timing attacks — NUNCA comparar hashes com !==
  const hashesMatch = crypto.timingSafeEqual(
    Buffer.from(expected),
    Buffer.from(config.password_hash)
  )
  if (!hashesMatch) {
    return NextResponse.json({ error: 'senha incorreta' }, { status: 401 })
  }

  // Cookie de auth — httpOnly, secure, SameSite=Lax, 24h
  const cookieVal = crypto.createHmac('sha256', secret)
    .update(params.token + config.password_hash)
    .digest('hex')

  const res = NextResponse.json({ ok: true })
  res.cookies.set(`dash_auth_${params.token}`, cookieVal, {
    httpOnly: true,
    secure: process.env.NODE_ENV === 'production',
    sameSite: 'lax',
    maxAge: 60 * 60 * 24,
    path: `/dashboard/${params.token}`,
  })
  return res
}
```
