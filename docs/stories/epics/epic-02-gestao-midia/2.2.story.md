### OAuth Flow — Google

```
Usuário clica "Conectar Google Ads"
  → GET /api/oauth/google/start
      ↳ state = crypto.randomUUID()
      ↳ cookies().set('oauth_state_google', state, { httpOnly, sameSite: 'lax', maxAge: 600 })
      ↳ redirect(googleOAuthUrl)  ← com access_type=offline, prompt=consent

Google OAuth UI
  → GET /api/oauth/google/callback?code=XXX&state=YYY
      ↳ valida cookie oauth_state_google === state param
      ↳ POST https://oauth2.googleapis.com/token { code, client_id, client_secret, redirect_uri, grant_type: 'authorization_code' }
      ↳ { access_token, refresh_token, expires_in }
      ↳ encryptToken(access_token, TOKEN_ENCRYPTION_KEY)   → encrypted_token
      ↳ encryptToken(refresh_token, TOKEN_ENCRYPTION_KEY)  → encrypted_refresh_token
      ↳ CustomerService.listAccessibleCustomers()          → customer_ids[]
      ↳ para cada customer: upsert em ad_accounts (platform='google')
      ↳ redirect('/settings/integrations?status=success&platform=google')
```

### refreshGoogleToken — Skeleton

```typescript
// apps/web/src/lib/oauth/google.ts
import { encryptToken, decryptToken } from '@advezo/utils'
import { createSupabaseServerClient } from '@advezo/database'

export async function refreshGoogleToken(
  adAccountId: string,
  encryptedRefreshToken: string,
  keyHex: string
): Promise<string> {
  const refreshToken = decryptToken(encryptedRefreshToken, keyHex)

  const res = await fetch('https://oauth2.googleapis.com/token', {
    method: 'POST',
    headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
    body: new URLSearchParams({
      grant_type: 'refresh_token',
      refresh_token: refreshToken,
      client_id: process.env.GOOGLE_CLIENT_ID!,
      client_secret: process.env.GOOGLE_CLIENT_SECRET!,
    }),
  })

  if (!res.ok) throw new Error(`Google token refresh failed: ${res.status}`)
  const { access_token } = await res.json()

  const newEncryptedToken = encryptToken(access_token, keyHex)

  const supabase = createSupabaseServerClient()
  await supabase
    .from('ad_accounts')
    .update({ encrypted_token: newEncryptedToken, status: 'active', error_message: null })
    .eq('id', adAccountId)

  return newEncryptedToken
}
```

### Observações (severidade BAIXA — não bloqueiam o gate)

- **CONCERN-1 (low / design — Story 2.4):** Em um cenário onde o Google **não** retorna `refresh_token` em uma re-autorização (improvável aqui, pois `prompt=consent` força o retorno), o upsert atual sobrescreveria `encrypted_refresh_token` com o novo valor — o que é o comportamento desejado nesta story. O código está correto para o fluxo de re-auth completa. A semântica de "preservar refresh_token antigo quando o novo não vier" só seria relevante se `prompt=consent` fosse removido no futuro; recomenda-se manter `prompt=consent` como invariante. Documentar em 2.4 se a política de consent mudar. **Sem ação requerida em 2.2.**
- **CONCERN-2 (low / observability):** `callback` faz `catch {}` silencioso (redireciona para `?error=google_oauth_failed` sem log estruturado). Aceitável para Wave 1 (não vaza token e dá feedback claro ao usuário), mas em Story 2.4/produção recomenda-se logging server-side correlacionado (sem o token) para diagnosticar falhas de OAuth. Espelha o padrão do callback Meta (2.1) — consistente. **Sem ação requerida em 2.2.**
