## Dev Notes

### Query de campanhas com gasto agregado

```typescript
// Server Component — apps/web/src/app/(dashboard)/campaigns/page.tsx
const supabase = createSupabaseServerClient()

// Busca ad_accounts para join manual (Supabase não suporta FK join com aggregate em uma query)
const { data: campaigns } = await supabase
  .from('ad_campaigns')
  .select(`
    id, platform, name, status, daily_budget, lifetime_budget,
    ad_account_id,
    ad_accounts!inner(account_name)
  `)
  .order('name', { ascending: true })
  // Aplicar filtros de platform/status via .eq() se presentes nos searchParams

// Gasto dos últimos 7 dias — query separada para agregação
const sevenDaysAgo = new Date(Date.now() - 7 * 24 * 60 * 60 * 1000).toISOString().split('T')[0]
const { data: spendData } = await supabase
  .from('campaign_metrics')
  .select('campaign_id, spend')
  .gte('date', sevenDaysAgo)

// Agregar spend por campaign_id no lado do cliente (server component)
const spendByCampaign = (spendData ?? []).reduce<Record<string, number>>((acc, row) => {
  acc[row.campaign_id] = (acc[row.campaign_id] ?? 0) + Number(row.spend)
  return acc
}, {})
```

## Dev Notes — Decisões de implementação (@dev)

- **[AUTO-DECISION] Ícone da sidebar → `Megaphone`** (AC 2.5.5 pedia `BarChart2`). Razão: `BarChart2` já está em uso por `/reports` (Relatórios) no `Sidebar.tsx` existente; reutilizá-lo criaria ambiguidade visual. O spawn do @pm instruiu explicitamente `Megaphone`. Desvio documentado e aprovado pela diretiva mais específica.
- **[AUTO-DECISION] Badge de status → inline no `CampaignRow`** em vez de reusar `StatusBadge`. Razão (IDS): o `StatusBadge` existente só aceita `'good' | 'warning' | 'critical'` (health do cliente), não os status de campanha `active/paused/deleted/archived`. ADAPT seria forçado; optei por um badge próprio reutilizando as mesmas classes `health-*` de design tokens para consistência visual.
- **[AUTO-DECISION] EmptyState com 3 variantes de subtitle**: sem contas → "Conecte uma conta…" (com action para `/settings/integrations`); com contas porém filtro sem match → "Nenhuma campanha corresponde aos filtros…"; com contas sem sync → "Aguardando próximo sync automático (06:00 UTC)". A 3ª variante (filtro) é um superset razoável do AC 2.5.4 sem inventar escopo.
- **IDS REUSE:** `PlatformIcon`, `EmptyState`, `Skeleton`, util `cn`, padrão de Server Component de `clients/page.tsx`, e o cálculo de `sevenDaysAgo` idêntico ao de `dashboard/page.tsx`.
- **Lint:** `react-hooks/purity` marca `Date.now()` como impuro mesmo em Server Component async (falso positivo, presente também no `dashboard/page.tsx` pré-existente). Suprimi com `eslint-disable-next-line` escopado e comentário explicativo, mantendo o novo arquivo lint-clean.
- **Build:** o gate simplificado do spawn (T6) roda TypeCheck + Testes (não há `turbo.json` no repo — comandos rodam direto em `apps/web`). `tsc --noEmit` passou limpo, servindo de proxy de corretude de build para esta Wave UI.
