- [x] **AC 2.1.2** — RLS em `ad_accounts`: `ALTER TABLE ad_accounts ENABLE ROW LEVEL SECURITY` + policy `workspace_isolation` via `auth_workspace_id()` (mesmo padrão da Story 1.4). - [x] **AC 2.1.4** — Route Handler `GET /api/oauth/meta/callback`: valida cookie `oauth_state` com o `state` recebido (CSRF guard — retorna 400 se diferente); troca `code` por `access_token` via `https://graph.facebook.com/oauth/access_token`; obtém token de longa duração via `https://graph.facebook.com/oauth/access_token?grant_type=fb_exchange_token`; criptografa o token com `encryptToken(accessToken, process.env.TOKEN_ENCRYPTION_KEY!)` de `@advezo/utils`; lista contas de anúncio disponíveis via `/me/adaccounts?fields=id,name,account_status`; persiste no banco; redireciona para `/settings/integrations?status=success`.

- [x] **AC 2.1.5** — Tokens NUNCA armazenados em texto puro: a coluna `encrypted_token` na tabela `ad_accounts` contém EXCLUSIVAMENTE o output de `encryptToken()` (formato `<iv_hex>:<authTag_hex>:<ciphertext_hex>`). Qualquer leitura para uso interno deve usar `decryptToken()` antes de passar para a API. Isso será verificado no QA Gate via inspeção do código.

- [x] **AC 2.1.6** — Suporte a múltiplas Business Managers por workspace: após OAuth, o callback lista todas as contas de anúncio acessíveis (`/me/adaccounts`) e persiste cada uma como uma linha separada em `ad_accounts`. - [x] **AC 2.1.9** — Variáveis de ambiente obrigatórias: `META_APP_ID`, `META_APP_SECRET`, `META_REDIRECT_URI` (ex: `${NEXT_PUBLIC_APP_URL}/api/oauth/meta/callback`), `TOKEN_ENCRYPTION_KEY` (64 hex chars = 32 bytes). ## Dev Notes

### DDL da Tabela `ad_accounts` (migration 000004)

```sql
CREATE TABLE ad_accounts (
  id                      uuid PRIMARY KEY DEFAULT uuid_generate_v4(),
  workspace_id            uuid NOT NULL REFERENCES workspaces(id) ON DELETE CASCADE,
  client_id               uuid REFERENCES clients(id) ON DELETE SET NULL,
  platform                text NOT NULL CHECK (platform IN ('meta', 'google')),
  external_account_id     text NOT NULL,
  account_name            text,
  encrypted_token         text NOT NULL,
  encrypted_refresh_token text,                 -- NULL para Meta; obrigatório para Google (Story 2.2)
  token_type              text DEFAULT 'access_token',
  status                  text NOT NULL DEFAULT 'active'
                            CHECK (status IN ('active', 'expired', 'error')),
  error_message           text,
  last_synced_at          timestamptz,
  created_at              timestamptz NOT NULL DEFAULT now(),
  updated_at              timestamptz NOT NULL DEFAULT now(),
  UNIQUE (workspace_id, platform, external_account_id)
);

ALTER TABLE ad_accounts ENABLE ROW LEVEL SECURITY;

CREATE POLICY workspace_isolation ON ad_accounts
  FOR ALL USING (workspace_id = auth_workspace_id());

CREATE TRIGGER ad_accounts_updated_at
  BEFORE UPDATE ON ad_accounts
  FOR EACH ROW EXECUTE FUNCTION trigger_set_timestamp();
```

### OAuth Flow Completo — Meta

```
Usuário clica "Conectar Meta Ads"
  → GET /api/oauth/meta/start
      ↳ crypto.randomUUID() → state
      ↳ cookies().set('oauth_state', state, { httpOnly, sameSite: 'lax', maxAge: 600 })
      ↳ redirect(metaOAuthUrl)

Meta OAuth UI (login + autorizar permissões)
  → GET /api/oauth/meta/callback?code=XXX&state=YYY
      ↳ cookies().get('oauth_state').value === state  ← CSRF guard
      ↳ fetch meta/oauth/access_token (code exchange)
      ↳ fetch meta/oauth/access_token?grant_type=fb_exchange_token (long-lived)
      ↳ fetch /me/adaccounts?fields=id,name,account_status (listar contas)
      ↳ para cada conta: encryptToken(accessToken, TOKEN_ENCRYPTION_KEY)
      ↳ upsert em ad_accounts (workspace_id do user autenticado)
      ↳ redirect('/settings/integrations?status=success')
```

### Uso do encryptToken

```typescript
// Em /api/oauth/meta/callback/route.ts
import { encryptToken } from '@advezo/utils'

const encryptedToken = encryptToken(accessToken, process.env.TOKEN_ENCRYPTION_KEY!)

await supabase.from('ad_accounts').upsert({
  workspace_id,
  platform: 'meta',
  external_account_id: account.id,
  account_name: account.name,
  encrypted_token: encryptedToken,
  status: 'active',
}, { onConflict: 'workspace_id,platform,external_account_id' })
```
