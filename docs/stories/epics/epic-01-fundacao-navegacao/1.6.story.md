## Story

**As a** gestor de tráfego,
**I want to** ver no dashboard principal um panorama de saúde de todos os meus clientes com indicadores coloridos (verde/âmbar/vermelho),
**so that** eu possa identificar de forma imediata quais clientes precisam de atenção sem abrir cada conta individualmente.

## Acceptance Criteria

- [x] **AC 1.6.1** — Rota `/(dashboard)/dashboard/page.tsx` (ou `/(dashboard)/page.tsx` com redirect) é a tela inicial após login. Título da página: "Dashboard".
- [x] **AC 1.6.2** — Barra de filtros no topo: select de "Todos os clientes" vs cliente específico, select de período (Hoje, 7 dias, 30 dias, 90 dias). Estado local com `useState`.
- [x] **AC 1.6.3** — 4 cards de métricas resumidas no topo: "Total de Clientes Ativos", "Clientes Saudáveis", "Clientes em Atenção", "Clientes Críticos". Cada card usa `HealthBar` ou `StatusBadge` da Story 1.3.
- [x] **AC 1.6.4** — Grid de `ClientHealthCard` abaixo dos métricas: um card por cliente ativo. Cada card mostra: avatar/logo, nome, `StatusBadge` com status de saúde, `HealthBar` com score numérico (0–100), e 2 métricas chave (ROAS e Gasto vs Budget — valores mockados nesta story).

- [x] **AC 1.6.6** — Lógica de cor de saúde: `score >= 70` → good (verde), `score >= 40` → warning (âmbar), `score < 40` → critical (vermelho). Usando tokens do design system da Story 1.3.
- [x] **AC 1.6.7** — Estado vazio: quando não há clientes ativos, mostrar `EmptyState` com `icon=Users`, `title="Nenhum cliente ativo"`, `subtitle="Cadastre clientes na seção Clientes para ver o dashboard"`, `action={{ label: "Ir para Clientes", href: "/clients" }}`.
- [x] **AC 1.6.8** — Loading state: enquanto dados carregam, mostrar skeleton (ShadCN `Skeleton`) nos cards de métrica e no grid. Sem layout shift.
- [x] **AC 1.6.9** — Dashboard é um Server Component que busca clientes via Supabase server client. `ClientHealthCard` é um Client Component apenas se precisar de interatividade.
- [x] **AC 1.6.10** — Testes: renderiza cards para cada cliente, renderiza EmptyState sem clientes, loading state correto, color mapping correto (score 75→good, 50→warning, 30→critical).

```
Dashboard
├── [Barra de Filtros]  ← DashboardFilters (Client Component)

│   ├── Em Atenção (40–69)
│   └── Críticos (< 40)

    ├── ClientHealthCard (cliente 2)
    └── ... ou EmptyState
```

```typescript
interface ClientHealthData {

}
```
