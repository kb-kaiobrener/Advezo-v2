## Dev Notes

### ShadCN base-nova e token layering (CRÍTICO — ler antes de implementar)

ShadCN 4.x (instalado na Story 1.1) usa o estilo **base-nova**, que opera com um sistema de tokens semânticos via CSS custom properties em `oklch()`:
```css
/* Gerado pelo ShadCN base-nova — já existe em globals.css */
--color-primary: oklch(0.205 0 0);   /* mapeado para bg-primary, text-primary etc. */
--color-secondary: oklch(...);
/* ... */
```

Os tokens Advezo (`brand-*`, `health-*`, `platform-*`) devem ser adicionados **em paralelo** a esse sistema, **não substituindo** as variáveis semânticas do ShadCN. A Strategy correta é:

```css
/* Em globals.css — adicionar JUNTO às variáveis ShadCN existentes, não no lugar */
:root {
  /* Tokens Advezo — namespace próprio, não sobrescreve ShadCN */
  --color-brand-600: #2563EB;
  --color-health-good: #16A34A;
  --color-health-warning: #D97706;
  --color-health-critical: #DC2626;
  /* ... */
}
```

No `tailwind.config.ts`, os tokens Advezo são adicionados como extensão. Os tokens ShadCN (`bg-primary`, `text-secondary` etc.) continuam funcionando para os componentes ShadCN nativos — sem conflito.

> **Nunca redefinir `--color-primary` para Advezo Blue** — isso quebra todos os componentes ShadCN que dependem de `bg-primary`. Use `brand-600` como nome de token próprio.

### tailwind.config.ts — Extensão Completa

Referência: `docs/ux-design-spec.md` Section 8 (código pronto para copiar).

```typescript
// Tokens a adicionar em theme.extend.colors:
brand: {
  50: '#EFF6FF',
  // ... blue-50 a blue-900
  600: '#2563EB',  // Advezo Blue — cor primária
  // ...
},
health: {
  good:     { DEFAULT: '#16A34A', text: '#15803D', bg: '#F0FDF4' },
  warning:  { DEFAULT: '#D97706', text: '#B45309', bg: '#FFFBEB' },
  critical: { DEFAULT: '#DC2626', text: '#B91C1C', bg: '#FEF2F2' },
},
platform: {
  meta:      '#1877F2',  // imutável — identidade da plataforma
  google:    '#4285F4',  // imutável
  whatsapp:  '#25D366',  // imutável
},
```

### Contraste WCAG AA verificado (docs/ux-design-spec.md Section 1.3)

| Token de texto | Ratio | Status |
|----------------|-------|--------|
| `text-green-700 (#15803D)` | 5.14:1 | ✓ WCAG AA |
| `text-amber-700 (#B45309)` | 4.74:1 | ✓ WCAG AA |
| `text-red-700 (#B91C1C)` | 4.99:1 | ✓ WCAG AA |
| `brand-600 (#2563EB)` sobre branco | 4.53:1 | ✓ WCAG AA |

Use sempre o token `text` do sistema de saúde, nunca o `DEFAULT` para texto sobre branco.

### HealthBar — Lógica de Cor

```typescript
function getHealthColor(value: number): string {
  if (value >= 70) return 'bg-health-good'
  if (value >= 40) return 'bg-health-warning'
  return 'bg-health-critical'
}
```
